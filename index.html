Flow هاوي vs Flow محترف

تأسيس الدافع — الفرق بين الهواة والمحترفين

في المحاضرات السابقة بنينا Flows شغّالة. بس فيه فرق كبير بين Flow "بيشتغل" وFlow "احترافي". النهاردة هنحوّل الـ Flows بتاعتك من المستوى الأول للتاني.

Flow بدون Conditions وError Handling = قنبلة موقوتة. هيشتغل كويس لحد ما يفشل — ووقتها مش هتعرف ليه ولا إزاي تصلحه.

Flow هاوي ❌

المشاكل

  • بيشتغل على كل إيميل (حتى Spam)
  • لو فشل → بيقف ومش بتعرف
  • محتاج تشغّله يدوياً
  • بيكرّر نفس الصف لو الإيميل اتبعت مرتين
  • مفيش فلتر على البيانات
Flow محترف ✅

المميزات

  • بيفلتر الإيميلات (Condition)
  • لو فشل → بيبلغك فوراً (Error Handling)
  • بيشتغل لوحده في وقت محدد (Scheduled)
  • بيتحقق من التكرار قبل ما يضيف
  • فيه validation على البيانات
القاعدة الذهبية

أي Flow فيه بيانات مهمة، لازم يكون فيه: Condition واحد على الأقل + Error Handling + Scheduled Trigger. ده اللي بيفرّق بين هاوي ومحترف.

٠٣

Conditions متقدمة

الدماغ اللي بيخلي الـ Flow يفكر — ١٥ دقيقة

الـ Condition هو "لو... إذن..." في الـ Flow. هو اللي بيخلي الـ Flow يتخذ قرارات ذكية بدل ما ينفذ أعمى.

Condition = دماغ الـ Flow
من غير Condition، الـ Flow بيشتغل أعمى. مع Condition، الـ Flow بيفكر قبل ما يتصرف.

مثال ١

فلتر المرسل

السيناريو: عايز الـ Flow يشتغل بس لو الإيميل من عميل مسجل، مش من Spam.
Condition: From contains "@customer.com" If Yes: → كمّل الـ Flow If No: → Terminate (تجاهل الإيميل)
✅ النتيجة: الـ Flow بيتجاهل أي إيميل مش من نطاق العملاء.
مثال ٢

فلتر الموضوع

السيناريو: عايز الـ Flow يتصرف differently لو الموضوع فيه "عطل طارئ".
Condition: Subject contains "عطل طارئ" If Yes: → ابعت SMS للمهندس المسؤول If No: → احفظ في Excel بس
✅ النتيجة: الأعطال الطارئة بتاخد معاملة أسرع.
مثال ٣

فلتر المبلغ

السيناريو: الفواتير فوق 10,000 درهم محتاجة موافقة المدير.
Condition: Amount greater than 10000 If Yes: → ابعت للمدير للموافقة If No: → كمّل عادي
✅ النتيجة: المبالغ الكبيرة بتاخد مراجعة إضافية.
نصيحة عملية

استخدم Conditions في 3 أماكن: 1. فلتر المرسل، 2. فلتر الموضوع/البيانات، 3. التحقق من صحة البيانات قبل الحفظ.

تدريب ١ — أضف Condition

٥ دقايق
  1. افتح Flow قديم بتاعك
  2. أضف Condition: Subject contains "عاجل"
  3. في If No: أضف Action "Terminate"
  4. اختبر بإيميل فيه "عاجل" وإيميل تاني مافيهوش
٠٤

Error Handling + Approval Action

الاحترافية في التعامل مع الأخطاء — ١٥ دقيقة

أي Flow في الدنيا ممكن يفشل. السؤال: لما يفشل، إنت هتعرف إزاي؟ والـ Flow هيعمل إيه؟

Error Handling مش عشان الـ Flow ما يفشلش — ده مستحيل.
Error Handling عشان لما يفشل، إنت تعرف وتصلح بسرعة.

مثال ٤

سيناريوهات الفشل الشائعة

السيناريوالسببالحل (Error Handling)
Excel مقفولمفتوح عند حد تانيابعت إشعار: "Excel مقفول — اقفله وأعد التشغيل"
API Key منتهيClaude API بيرفضابعت إيميل للمدير: "API Key محتاج تجديد"
بيانات ناقصةالإيميل مافيهوش رقم الحسابابعت رد للعميل: "نعتذر، نحتاج رقم الحساب"
OneDrive ممتلئمفيش مساحةابعت إشعار: "OneDrive ممتلئ — نظّف الملفات القديمة"
Network Errorالإنترنت قطعRetry بعد 5 دقايق (Automatic Retry)
القاعدة الذهبية

كل Action مهم في الـ Flow، لازم يكون فيه Error Handling. لو Action فشل ومش معالج، الـ Flow كله هيوقف.

مثال ٥

تطبيق Error Handling في Power Automate

السيناريو: لو Action "Add row to Excel" فشل، ابعت إشعار للمدير.
  1. بعد Action "Add row to Excel"، اضغط "Add an action"
  2. اختار: Configure run after
  3. علّم على: "has failed" و "has timed out"
  4. أضف Action: Send an email (V3)
  5. في Body: اكتب "Flow فشل في خطوة Add row. راجع Run History."
✅ النتيجة: لو الـ Action فشل، المدير هياخد إيميل فوراً.

Approval Action — للموافقات الإدارية

Approval Action للشركات الحكومية

المثال: فاتورة جديدة فوق 10,000 درهم
الـ Flow:
1. Trigger: فاتورة جديدة
2. Action 1: ابعت للمدير للموافقة
3. Action 2: لو المدير وافق → احفظ في Excel
           لو المدير رفض → ابعت رد للعميل

💡 ده مهم جداً في الشركات الحكومية — أي معاملة مالية محتاجة موافقة.

تدريب ٢ — أضف Error Handling

دقايق
  1. افتح Flow قديم
  2. بعد أي Action مهم، أضف "Configure run after"
  3. علّم على "has failed"
  4. أضف Action: Send email notification
  5. اختبر: عطّل الـ Action عمداً وشوف الإشعار
٠

Scheduled Flows + Solution Sharing

الأتمتة الزمنية + مشاركة الـ Flow — ١٥ دقيقة

آخر نوع من الـ Flows: Scheduled. ده اللي بيشتغل في وقت محدد — حتى لو إنت في إجازة. ده الهدف النهائي من الكورس.

Scheduled Flow = حرية حقيقية
مش محتاج تفتح اللابتوب. مش محتاج تتذكر. الـ Flow بيشتغل لوحده في الوقت اللي حددته.

مثال ٦

تقرير يومي تلقائي

السيناريو: كل يوم 8 الصبح، المدير عايز تقرير بالأمس.
Trigger: Recurrence Every day at 8:00 AM Actions: 1. Get rows from Excel (أمس) 2. Claude: اعمل ملخص 3. Create file: Daily_Report_[Date].pdf 4. Send email to manager with attachment
✅ النتيجة: المدير بيستلم تقرير كل يوم 8 الصبح — حتى لو إنت في إجازة.
مثال ٧

تنظيف أسبوعي

السيناريو: كل جمعة 6 العصر، نظّف الملفات القديمة.
Trigger: Recurrence Every Friday at 6:00 PM Actions: 1. Get files from OneDrive (أقدم من 30 يوم) 2. Move to Archive folder 3. Send notification: "تم الأرشفة"
✅ النتيجة: OneDrive دايماً نظيف ومنظم.

🆕 Solution Sharing — مشاركة الـ Flow

إزاي تشارك الـ Flow مع زملائك؟

الطريقة 1: Export/Import
1. افتح الـ Flow → Export → احفظ كـ .zip
2. ابعت الملف للزميل
3. الزميل: Import → عدّل الـ Connections

الطريقة 2: Copy to My Flows
1. افتح الـ Flow → Share → Copy to My Flows
2. الزميل يلاقيه في My Flows بتاعه

💡 الطريقة 2 أسهل لكن محتاج صلاحيات Shared

نصيحة عملية

استخدم Scheduled Flows للمهام الدورية: تقارير يومية، تنظيف أسبوعي، تذكيرات شهرية. ده اللي بيوفرلك ساعات كل أسبوع.

تدريب ٣ — ابنِ Scheduled Flow

٥ دقايق
  1. في Power Automate: Create → Scheduled cloud flow
  2. اضبط: Every day at 8:00 AM
  3. أضف Action: Send yourself an email "Good morning!"
  4. اختبر: استنى لحد ما الـ Flow يشتغل

المشروع 1: Flow شكاوى العملاء الكامل

أول مشروع احترافي — ٢٥ دقيقة

دلوقتي هنبنّي أول مشروع احترافي: Flow شكاوى العملاء. ده Flow كامل فيه كل اللي اتعلمناه: Conditions + Error Handling + AI + Excel.

1
Flow شكاوى العملاء الاحترافي

الهدف: كل شكوى توصل → AI يحللها → تحفظ في Excel → إشعار للموظف المسؤول. كل ده أوتوماتيك.

الخطوة 1: Trigger

  1. When a new email arrives (Outlook)
  2. Folder: Inbox
  3. Subject Filter: شكوى

الخطوة 2: Condition 1 — فلتر المرسل

  1. Condition: From contains "@customer.com"
  2. If Yes → كمّل
  3. If No → Terminate (تجاهل)

الخطوة 3: HTTP Action — Claude

POST https://api.anthropic.com/v1/messages Headers: x-api-key: [API Key] anthropic-version: 2023-06-01 Body: { "model": "claude-3-5-sonnet-20241022", "max_tokens": 1024, "messages": [{ "role": "user", "content": "استخرج: رقم الحساب، نوع العطل، الملخص. JSON فقط. الإيميل: @{triggerOutputs()?['body/body']}" }] }

الخطوة 4: Parse JSON

  1. Parse JSON Action
  2. Content: Body من HTTP Action
  3. Schema: Generate from sample

الخطوة 5: Condition 2 — التحقق من البيانات

  1. Condition: رقم الحساب is not empty
  2. If Yes → كمّل
  3. If No → Send email to customer: "نعتذر، نحتاج رقم الحساب"

الخطوة 6: Add row to Excel

  1. Add row to Complaints_Log.xlsx
  2. Columns: التاريخ، من، رقم الحساب، نوع العطل، الملخص، الحالة = "جديد"

الخطوة 7: Error Handling

  1. بعد Add row: Configure run after → "has failed"
  2. Action: Send email to admin: "Flow فشل — راجع Run History"

الخطوة 8: Notification

  1. Send mobile notification: "شكوى جديدة من [العميل] — نوع العطل: [النوع]"
✅ النتيجة: Flow احترافي كامل. فيه Conditions، Error Handling، AI، Excel، Notifications. ده Flow محترف.

تدريب ٤ — ابنِ المشروع 1

٢٠ دقيقة
  1. اتبع الخطوات من 1 لـ 8
  2. اختبر بإيميل تجريبي فيه "شكوى"
  3. تأكد إن البيانات دخلت Excel صح
  4. اختبر Error Handling: عطّل Excel وشوف الإشعار

المشروع 2: Flow التقرير الأسبوعي

ثاني مشروع احترافي — ٢٠ دقيقة

المشروع التاني: Flow التقرير الأسبوعي. ده Flow بيجمع كل حاجة في تقرير واحد للمدير.

2
Flow التقرير الأسبوعي الشامل

الهدف: كل جمعة 5 العصر → اجمع بيانات الأسبوع → Claude يعمل تقرير → ابعت للمدير.

الخطوة 1: Trigger

  1. Recurrence: Every Friday at 5:00 PM

الخطوة 2: Get Data

  1. Get rows from Complaints_Log.xlsx (آخر 7 أيام)
  2. Get rows from Meter_Readings.xlsx (آخر 7 أيام)

الخطوة 3: Claude Report

HTTP Action to Claude Prompt: "اعمل تقرير أسبوعي مختصر من البيانات دي: عدد الشكاوى، أنواع الأعطال، الاستهلاك الكلي، التحذيرات. ارجع Word document."

الخطوة 4: Save & Send

  1. Create file: Weekly_Report_[Date].docx
  2. Send email to manager with attachment
✅ النتيجة: المدير بيستلم تقرير أسبوعي كل جمعة 5 العصر — أوتوماتيك.

تدريب ٥ — ابنِ المشروع 2

١٥ دقيقة
  1. اتبع الخطوات من 1 لـ 4
  2. اختبر: شغّل الـ Flow يدوياً (Run now)
  3. تأكد إن التقرير اتعمل واتبعت

الواجب النهائي + إغلاق

تطبيق شامل — آخر ١٠ دقايق

التدريب الختامي — 2 Projects شغّالة

١٠ دقايق
  1. تأكد إن الـ 2 Projects شغّالة
  2. اختبر كل Flow بإيميل/بيانات تجريبية
  3. تأكد إن Conditions شغالة
  4. تأكد إن Error Handling شغال
  5. احفظ Screenshots من Run History

الواجب النهائي — سلّميه خلال أسبوع

  • طبّق الـ 2 Projects على شغلك الفعلي (عدّل الـ Prompts والـ Conditions)
  • اختبر كل Flow بـ 5 إيميلات/بيانات حقيقية
  • صور الـ Run history لكل Flow (نجح/فشل)
  • ابعت الـ 2 Projects شغّالة على WhatsApp للمراجعة
  • اكتب ملخص: إيه الوقت اللي وفرته في كل Flow (دقايق/أسبوع)

أسئلة المراجعة الذاتية — نظرية

١. ما الفرق بين Flow هاوي وFlow محترف؟
٢. ليه Conditions مهمة في أي Flow؟
٣. ما هو Error Handling وليه ضروري؟
٤. ما الفرق بين Automated Flow وScheduled Flow؟
٥. إيه الـ 2 مشاريع اللي بنيناهم في المحاضرة دي؟

أسئلة المراجعة الذاتية — عملية

١. أضف Condition لـ Flow قديم: Subject contains "عاجل".
٢. أضف Error Handling لـ Flow: لو فشل، ابعت إشعار.
٣. ابنِ Scheduled Flow: كل يوم 8 الصبح ابعت إيميل لنفسك.
٤. طبّق المشروع 1 (شكاوى العملاء) على شغلك الفعلي.
٥. احسب الوقت اللي وفرته الـ 2 Projects في الأسبوع.
